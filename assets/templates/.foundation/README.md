这里是本项目的 AI Coding 治理地基。它不负责写 PRD、不生成 Specs、不制定项目计划；它负责让后续 agent 的工作有原则、有状态、有记录、有检查、有待裁决建议。

## 文件职责

| 文件 | 作用 | 更新时机 |
|---|---|---|
| `AGENTS.md` | agent 工作入口，定义 Mandatory Loop、Hard Rules 和冲突处理 | 入口协议、工作循环或硬规则需要调整时 |
| `PRINCIPLES.md` | 长期原则、禁区和取舍依据 | 信息具备长期约束意义且裁决清晰时 |
| `STATE.md` | 当前接续快照，回答“现在接手要知道什么” | 当前状态、焦点、阻塞或接续点变化时 |
| `LOG.md` | 历史账本，记录重要事件 | 发生重要变更、决策、事故、验收或复盘时 |
| `CHECKS.md` | 检查清单，用于执行前、执行中、完成后自检 | 检查项、验证习惯或误投扫描需要调整时 |
| `SUGGESTIONS.md` | 未裁决治理提案队列，覆盖 `AGENTS.md` 和 `.foundation/` 下任一治理文件 | AI 自发发现治理文件需要演进但尚未获得裁决时 |

## 记忆路由表

本文档是唯一记忆路由权威。其它治理文件可以引用或检查本表，不应重新定义路由。

| 信息性质 | 倾向写入 |
|---|---|
| 入口协议、工作循环、硬规则、冲突处理 | `AGENTS.md` |
| 长期约束、禁区、稳定偏好、取舍依据 | `PRINCIPLES.md` |
| 当前恢复所需状态、工作焦点、阻塞、接续点 | `STATE.md` |
| 已发生的重要事件、决策、事故、验收、复盘 | `LOG.md` |
| 执行前后检查项、验证习惯、误投扫描 | `CHECKS.md` |
| AI 自发发现但尚未裁决的治理改进 | `SUGGESTIONS.md` |

记录前先判断信息性质。不确定该写到哪里时，先说明判断并请用户裁决；不要把所有信息都塞进同一个文件。

## 读取规则

- 每轮开工前读：`AGENTS.md`、本文件、`PRINCIPLES.md`、`STATE.md`。
- 重大改动前读：`CHECKS.md`。
- 发生变更、事故、迭代或复盘后，按记忆路由表判断是否记录。
