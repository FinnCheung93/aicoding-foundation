这里是本项目的 agentic 治理地基。它不替代需求文档、执行方案或项目计划；它负责让后续 agent 的工作有入口、原则、当前状态、历史记录和未裁决治理提案。

## 文件职责

| 文件 | 作用 | 更新时机 |
|---|---|---|
| `AGENTS.md` | agent 工作入口，保留最低启动动作、关键触发和硬底线 | 入口协议、关键触发或硬规则需要调整时 |
| `PRINCIPLES.md` | 长期原则、禁区和取舍依据 | 信息具备长期约束意义且裁决清晰时 |
| `STATE.md` | 当前接续快照，回答“现在接手要知道什么” | 当前目标、状态、焦点、阻塞或接续点变化时 |
| `LOG.md` | 重要历史账本，记录重要变更、决策、事故、验收和复盘 | 发生值得追溯的重要事件时 |
| `SUGGESTIONS.md` | 未裁决治理提案队列，也承接防复发候选 | 治理改进需要用户裁决后才能生效时 |

## 信息路由

本文档是唯一记忆路由权威。其它治理文件可以引用本表，不应重新定义路由。

| 信息性质 | 倾向写入 |
|---|---|
| 入口协议、关键触发、硬规则、冲突处理 | `AGENTS.md` |
| 长期约束、禁区、稳定偏好、取舍依据 | `PRINCIPLES.md` |
| 当前恢复所需目标、状态、焦点、阻塞、接续点 | `STATE.md` |
| 已发生的重要变更、决策、事故、验收、复盘 | `LOG.md` |
| 未裁决治理改进、防复发候选 | `SUGGESTIONS.md` |

记录前先判断信息性质。不确定该写到哪里时，先说明判断并请用户裁决；不要把所有信息都塞进同一个文件。

## 读取判断

不要默认读取所有文档。先判断缺失某类信息是否会造成误判、越权、覆盖用户内容、重复已拒绝方向、无法接续或无法验证。

- 缺少当前目标、状态、阻塞或下一步会影响接续时，读取 `STATE.md`。
- 缺少长期原则、禁区、稳定偏好或取舍依据会影响判断时，读取 `PRINCIPLES.md`。
- 涉及历史裁决、事故、验收、复盘、不做决定或避免重复建议时，读取 `LOG.md` 的相关记录。
- 涉及未裁决治理变更、防复发候选或治理队列处理时，读取 `SUGGESTIONS.md`。
- 涉及专业流程、项目规范、架构、需求、运行手册或 QA / 验收 / 发布检查时，读取下方登记的扩展文档；未登记且缺失会阻塞时先对齐。

## 记录证据门槛

- 只有会影响后续判断、行动、恢复或治理演进的信息才沉淀。
- 普通聊天、临时探索过程、已被其它 artifact 完整承载的长内容，不重复写入治理文件。
- 引用优先：已有正式 artifact 或外部文档能承载的信息，治理文件只写路径、文件名或简短摘要。
- 用户明确裁决“拒绝 / 不做 / 暂不支持 / 不要再建议”的方向时，普通裁决进入 `LOG.md`，长期禁区或稳定取舍进入 `PRINCIPLES.md`，未裁决的治理改进进入 `SUGGESTIONS.md`。

## 防复发提案入口

agent 出错后的重点不是“错误放哪个文件”，而是判断是否暴露可复发的治理缺口。

进入 `SUGGESTIONS.md` 的防复发候选应同时满足：

- 可能复发。
- 会影响后续 agent 判断、行动或治理演进。
- 暴露了入口协议、信息路由、长期原则、状态接续或治理触发机制的缺口。
- 需要用户裁决后才能进入正式治理文件。

不满足这些条件的普通错误、当场修好的问题、临时执行过程，不进入 `SUGGESTIONS.md`。

## 扩展项目文档

项目可以按需维护架构、领域、开发约定、决策、需求、操作手册、QA / 验收 / 发布检查、Agent 专用适配等扩展文档；这些文档不由本地基默认创建。新增后应在下表登记职责、依赖性质和读取时机，避免散落或与通用入口冲突。

| 文档 / 目录 | 职责 | 依赖性质 | 何时读取 |
|---|---|---|---|
| 待补充 | 待补充 | 硬依赖 / 软依赖 | 待补充 |

扩展规则：

- `AGENTS.md` 只做短入口和索引，不承载完整业务、架构、开发规范或操作手册。
- 硬依赖是缺失会导致任务错误、越权或无法验证的资料；缺失时先对齐。软依赖是有助于提高质量但不阻塞执行的资料；缺失时继续工作，必要时记录改进建议。
- 优先维护通用入口：`README.md`、`AGENTS.md`、`.foundation/`。Agent 专用适配层不应形成另一套冲突规则。
- 团队级、语言级、平台级的大型规范不在项目内全文复制；项目内只保留摘要、链接和本项目例外。
- 共享语言文档可以作为软依赖登记，用于统一项目术语和避免混用；它不承载 spec、实现细节或计划。
- 重要长期决策按现有路由沉淀到 `LOG.md`、`PRINCIPLES.md` 或 `SUGGESTIONS.md`；不默认创建 ADR 文件夹。
- Foundation 只负责登记和路由，不接管 PRD、Specs、任务拆分、架构设计、QA 或其它专业流程。
