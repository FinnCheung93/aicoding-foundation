# Suggestions

本文档是未裁决治理提案队列，不是普通建议箱。

它服务整个治理地基的自进化。只记录 AI 自发发现、尚未被用户裁决的治理变更提案；已裁决且应立即生效的内容应进入对应正式治理文件，并由 `LOG.md` 记录事件。

## 提案格式

```text
时间：
建议目标文件：AGENTS.md / README.md / PRINCIPLES.md / STATE.md / LOG.md / CHECKS.md / SUGGESTIONS.md
建议类型：触发规则 / 记录规则 / 恢复规则 / 检查规则 / 原则变更 / 模块边界 / 结构调整 / 验证补强 / 评审团机制
触发事件：
建议内容：
原因和证据：
影响范围：
用户裁决问题：
状态：待确认 / 已确认 / 已拒绝 / 已执行
```

## 使用规则

- 只有未裁决的治理变更进入本文档。
- 提案可以指向 `AGENTS.md` 或 `.foundation/` 下任一治理文件。
- 用户确认后，才能修改正式治理文件。
- 已执行的治理变更应记录到 `LOG.md`。

## 队列处理

- 重要工作收尾、用户询问治理问题、或版本同步前，快速查看是否有待确认提案影响本轮工作。
- 用户确认提案后，把变更增量写入目标治理文件，并将提案状态改为“已执行”。
- 用户拒绝提案后，将状态改为“已拒绝”，保留原因摘要即可。
- 已执行或已拒绝的提案移入“已处理提案”；“待裁决提案”只保留待确认或已确认但未执行的内容。
- 多条提案冲突或重叠时，先合并判断或请求用户裁决，不要分别硬改。
- 不把本文档当作长期规则来源；正式生效内容应进入对应治理文件。
