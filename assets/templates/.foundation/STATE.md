计数规则：

- 只有出现合格治理信号时才计数 +1；普通聊天、正常执行、短澄清和日常状态更新不计数。
- 合格治理信号包括：重复协作摩擦、规则冲突、用户明确纠偏、agent 因治理规则犹豫或误判、事故/返工暴露规则缺口、同类建议重复出现。
- 计数达到 3 / 3 时，在自然停顿点做一次轻量治理复盘，然后将计数重置为 0 / 3。
- 自然停顿点包括：任务收尾、开始新阶段前、用户询问治理、准备修改 `AGENTS.md` 或 `.foundation/` 前。
- 如果任务仍在推进中，只记录待复盘信号，不强制打断；但开启新阶段前不得跳过已到期复盘。
- 复盘结论只在三类中选择：保留现状、修改对应正式治理文件、写入 `SUGGESTIONS.md` 等待裁决。
